为什么以上操作符不可以被overload？
?:操作符不能被overload，因为没有必要对三元操作符overload，重载之后的表达式expr1?expr2:expr3不能保证expr2和expr3中只有一个被执行。
sizeof不可以被overload，因为它是内置的操作符，对一个指向数组的指针进行增加会隐式的调用该操作符，如下程序：
X a[10];
X* p = &a[3];
X* q = &a[3];
p++; // p points to a[4]
// thus the integer value of p must be
// sizeof(X) larger than the integer value of q
因此，程序员如果不违反基本的语言规则，既不能够为sizeof(X)提供一个全新且不同的意义。
对于N::m，N和m都不是含有值的表达式，，对于编译器来说，N和m都只是名字，::在编译阶段提供了范围的解释而不是表达式的解析。
.(dot)操作符原则是可以按照和->操作符相同的技术进行重载的，但是这样做会产生歧义，到底是.的重载还是引用？如下例子：

需要注意的地方：
1.为了使操作符重载有效，至少有一个操作数是自定义类的对象。
2.赋值操作符，编译器会对每一个类自动创建一个默认的赋值操作符，行为和拷贝构造函数相同。
3.转换操作符，可以实现转换操作符，将一个类型转化为另一个类型。
