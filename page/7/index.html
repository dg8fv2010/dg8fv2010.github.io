GeeksforGeeks 013-C++中的inline函数

By Xue Wentao

发表于 2015-10-07

inline函数会被编译器在编译期间替换

编译器可能会忽视inline：

1. 函数包含循环
2. 函数包含static关键字
3. 递归函数
4. 函数的返回类型不是void，并且函数体内没有return
5. 函数包含switch或goto

优点：

1. 不会发生Function call overhead
2. 当函数被调用时，省去了栈上的操作
3. 省去了函数的return调用
4. 可以让编译器执行一些优化
5. 适合嵌入式系统

缺点：

1. 在inlined函数里面增加的变量会占用额外的register
2. 使用太多inline函数会使二进制文件变大
3. 太多的inline函数会降低instruction cache hit rate
4. 增加编译时间
5. 对嵌入式系统可能不管用，因为对于嵌入式系统代码大小比速度更重要
6. 二进制文件的增加可能导致thrashing
