为什么以上操作符不可以被overload？
?:操作符不能被overload，因为没有必要对三元操作符overload，重载之后的表达式expr1?expr2:expr3不能保证expr2和expr3中只有一个被执行。
sizeof不可以被overload，因为它是内置的操作符，对一个指向数组的指针进行增加会隐式的调用该操作符，如下程序：
X a[10];
X* p = &a[3];
X* q = &a[3];
p++; // p points to a[4]
// thus the integer value of p must be
// sizeof(X) larger than the integer value of q
因此，程序员如果不违反基本的语言规则，既不能够为sizeof(X)提供一个全新且不同的意义。
对于N::m，N和m都不是含有值的表达式，，对于编译器来说，N和m都只是名字，::在编译阶段提供了范围的解释而不是表达式的解析。

看以来很奇怪。但是析构函数往往按照类派生相反的顺序被调用。这就意味着派生类的析构函数会先被调用，然后基类的析构函数才会被调用。
如果纯虚析构函数的函数体没有定义，那么在对象析构的过程中就会出错。

GeeksforGeeks-052-虚函数可以是inline的吗？

By Xue Wentao

发表于 2015-11-14

虚函数用于实现运行时多态，又称为动态绑定或后置绑定。inline函数出于效率而使用。
inline函数背后的思想是，无论何时inline函数被调用时，在编译阶段，inline函数里面的代码会在该inline函数被调用处插入或替换。
当小函数经常被使用，并且在程序中被调用了很多次的时候，inline函数非常有用。

在默认情况下，所有在类里面的函数都会隐式或自动地被视为inline函数，虚函数除外。

无论何时，如果使用基类的指针或引用来调用虚函数时，它不可能是inline的（因为调用时在运行时被解释的）。
但是，在使用这个类的对象调用时，可以是inline的，因为编译器在编译阶段就可以准确地知道类的对象。
