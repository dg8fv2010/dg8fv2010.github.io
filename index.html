GeeksforGeeks-080-C++中的const成员函数

By Xue Wentao

发表于 2015-11-14

const函数的想法是不允许修改他们调用的对象。如果可能，让尽量多的函数称为const函数，来避免对象的以外修改。

#include
using namespace std;

class Test {
int value;
public:
Test(int v = 0) {value = v;}

// We get compiler error if we add a line like "value = 100;"
// in this function.
int getValue() const {return value;}

};

int main() {
Test t(20);
cout<<t.getValue();
return 0;
}
Output:
20

当一个函数被声明为const，它可以被任何类型的对象调用。非const函数只能被非cosnt对象调用。

#include
using namespace std;

class Test {
int value;
public:
Test(int v = 0) {value = v;}
int getValue() {return value;}
};

int main() {
const Test t;
cout << t.getValue();
return 0;
}
Output:
passing ‘const Test’ as ‘this’ argument of ‘int
Test::getValue()’ discards qualifiers

模板特化是怎样工作的？
当我们在实现一个类或函数的模板时，当被一个新的数据类型或新的数据类型集合（以防多个模板参数）使用时，编译器都会创建一份类或函数的拷贝。
当一个特化版本出现时，编译器首先检查特化版本，再检查主模板。
Compiler first checks with the most specialized version by matching the passed
parameter with the data type(s) specified in a specialized version.
但出现一个特化版本时，编译器首先检查特化版本再检查主模板。编译器会首先会对传入参数的类型进行匹配。

GeeksforGeeks-074-C++中的模板元编程

By Xue Wentao

发表于 2015-11-14

模板元编程：template metaprogramming
预测下面程序的输出：

Output:

程序计算2的8次方。
实际上，funStruct可以用来计算任何已知类型n的2^n。以上程序需要注意的是：计算是在编译阶段完成的。因此，实际上是编译器计算的2^8。
为了理解编译器是如何实现的，需要考虑模板和枚举的情况。

  1. 我们可以向模板类/函数传递非类型参数（不是任何数据类型的参数）。
  2. 和其他常量表达式一样，枚举常量的值会在编译阶段被计算。
  3. 当编译器发现了一个模板的新参数时，编译器会创建模板的新实例。

观察原始程序，当编译器看到funStruct<8>::val时，会试图创建一个以8作为参数的funStruct实例。然后发现funStruct<7>也需要作为枚举常量被创建，枚举常量的值会在编译阶段被计算。
对于funStruct<7>，编译器需要funStruct<6>并以此类推。最后编译器使用funStruct<1>::val，在编译阶段递归终止。
因此，通过使用模板，我们可以编写在编译阶段就进行计算的程序，这样的程序叫做模板元程序（template metaprogramming）。
模板元编程实际上是Turing-complete，意思就是任何可以用程序表达的数学式都可以被计算。
模板元编程通常不会在实际编程中使用。

函数重载和模板有什么区别？
函数重载和模板都是OOP中多态的体现，函数重载是多个函数执行相似的操作，模板是多种函数执行同一个操作。

模板怎么工作的？
模板在编译时被扩展。和宏类似。不同的地方是，编译器在模板扩展时会进行类型检查。源代码只包含函数/类，但是编译后的代码可能包含多个相同函数/类的拷贝。

什么是模板特化/特殊化/定制
template specialization：允许我们可以针对特定的数据类型拥有不同的代码。
